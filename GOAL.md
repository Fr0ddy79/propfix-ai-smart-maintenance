# GOAL.md — PropFix AI

## Vision
Build **PropFix AI** into the simplest and most effective maintenance coordination platform for small and mid-sized property managers.

The product should help property managers handle the full maintenance workflow with less admin work, faster response times, better tenant communication, and cleaner contractor coordination.

The long-term vision is to become the operating system for maintenance operations:
- tenant request intake
- AI-powered triage
- contractor dispatch and scheduling
- status tracking and communication
- reporting and operational visibility

The product must feel:
- fast
- clear
- trustworthy
- operational
- easy to use without training

---

## Core Product Goal
Create a SaaS web app that makes this workflow effortless:

**Tenant submits issue → manager reviews → contractor executes → tenant confirms completion**

This is the core loop.  
Everything in the product should support, accelerate, or simplify this loop.

---

## Primary Objective
Launch a polished MVP that is good enough to:
1. demo to early users
2. validate demand with pilot customers
3. support first paying property managers
4. serve as the base for future AI automation and deeper product development

---

## Target Users

### Primary Buyer
**Property managers** managing roughly 20–250 units.

### Secondary Users
**Tenants**
- submit requests
- view status
- confirm completion

**Contractors**
- receive assigned jobs
- update progress
- upload completion proof

---

## UX / UI Direction

### UX Principles
- One screen should focus on one main decision
- Important actions should be reachable in under 3 clicks
- Reduce friction and typing wherever possible
- Make next steps obvious at all times
- Keep onboarding simple and fast
- Prioritize operational clarity over visual novelty

### UI Principles
- Modern B2B SaaS look
- Clean layouts with strong hierarchy
- Soft backgrounds, rounded cards, subtle shadows
- Blue as primary accent
- Status colors only where useful
- Accessible contrast and responsive design
- Product should feel premium but efficient

### Product Feeling
The app should feel like a mix of:
- Linear for clarity
- Stripe for trust
- Notion for calm structure
- modern dashboard SaaS for speed and usefulness

## MVP Scope

### Must-Have
- Marketing website
- Pricing page / section
- FAQ / trust-building content
- Property manager dashboard
- Tickets list
- Ticket detail page
- Calendar / schedule view
- Contractors page
- Settings page
- Tenant request submission portal
- Contractor jobs portal

### Core MVP Features
- maintenance request intake
- ticket creation
- basic AI triage display
- contractor assignment
- scheduling
- status tracking
- message/activity thread
- manager action center
- onboarding hints / empty states

### AI Feature for MVP
The main AI feature should be:

**Unstructured request → structured ticket**

Example:
- tenant writes: “Kitchen sink is leaking under the cabinet and it’s getting worse”
- AI produces:
  - category: Plumbing
  - priority: High
  - urgency: Urgent
  - suggested trade: Plumber

AI should support clarity and speed, not overwhelm the UI.

---

## Out of Scope for Initial MVP
Do not overbuild these yet:
- predictive maintenance
- IoT integrations
- advanced reporting
- complex marketplace features
- deep billing workflows
- overly complex analytics
- enterprise-only workflows

These can come later once the core loop is validated.

## Product Standards
Every feature should pass these checks:
1. Does it support the core maintenance loop?
2. Does it reduce friction for the manager?
3. Does it make the UI clearer, not heavier?
4. Does it improve speed, visibility, or trust?
5. Is it necessary for MVP, or should it wait?

If the answer is no, it should probably not be built yet.

---

## Success Metrics
The project is moving in the right direction if:
